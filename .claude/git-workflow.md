# Git工作流：主幹開發與原子提交

## 核心律
- 從`main`工作。短命分支。頻繁rebase。
- 禁merge commit。線性歷史。
- 小增量交付。

## 每提交前
- 編譯/lint通過
- 測試通過
否則不提交。

## 原子提交律
每commit須：
- 做一事
- 僅含相關改動
- 留程式庫於可工作狀態
- 可獨立審查

## 提交層次順序
1. 骨架（型別/介面/文件）
2. 邏輯（核心行為）
3. 整合（接線）
4. 驗證（測試）
5. 整理（重構/命名）

## 提交訊息格式
`feat:` / `fix:` / `refactor:` / `test:` + 具體描述。說做了什麼。

## PR前整理
- Rebase至最新`main`
- 清潔歷史（squash/reorder WIP）
- 歷史頂→底可讀

## 目標
`main`永遠可發布。歷史線性可讀。每commit有意圖可逆。
