# 測試契約

測試=規格非障礙。八律：

1. **禁改既有測試使其通過。** 先前通過測試→你的改動後失敗=你的改動破壞預期行為。修碼非修測。唯例：明言改變該測試所描述行為，且預先聲明。

2. **禁弱化斷言。** 以寬斷言換窄使失敗測試通過=永不可接受。

3. **禁刪測試解故障。** 刪測試=刪規格。標記討論，不默默刪除。

4. **禁改既有函數行為滿足新測試。** 功能B之測試揭示函數X需不同行為→停止評估。正途：加新函數或參數，非悄改X。

5. **新功能致既有測試失敗→舉證責任在新功能。** 新碼必須整合而不破壞既有行為。

6. **若認為既有測試確實錯誤**→加注釋+問確認，方可改動。

7. **收到錯誤訊息或bug**→先寫預期行為之失敗測試→再修使其通過。

8. **找根因**，非繞過錯誤訊息最短路。直接解決底層問題。

測試組=棘輪，唯進。
